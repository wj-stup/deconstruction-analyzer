# 搜索策略 SOP

> 情报搜集阶段（Step 2）的标准操作流程
> 更新时间：2026-07-05

---

## 1. 搜索关键词组合模板

按搜索轮次分 3 组，每组最多 2 个并行请求：

### 第一轮：类型+平台定位

```
"{类型关键词} 小说 {平台} 热门 推荐"
"{类型关键词} 网文 排行 2025 2026"
```

### 第二轮：对标作品深挖（每本作品一个查询）

```
"{作品名} 小说 评价 读者 评论"
"{作品名} 小说 结构 分析 开篇 节奏"
```

### 第三轮：写作技法补充

```
"{类型关键词} 开篇 黄金三章 写法 结构"
"{类型关键词} 常见问题 雷区 避坑"
```

## 2. 平台优先级

| 优先级 | 平台 | 适用类型 | 搜索侧重 |
|:-------|:-----|:---------|:---------|
| 1 | 番茄小说 | 男频囤货/末世/都市 | 作品页简介+在读数+热评 |
| 2 | 起点中文网 | 玄幻/修仙/男频 | 作品信息+推荐票+章节目录 |
| 3 | 七猫小说 | 女频/言情 | 免费阅读受众 |
| 4 | 起点女生网 | 女频末世/言情 | 女频细分 |

## 3. 每部对标作品最低信息标准

| 信息类别 | 必需 | 示例 |
|:---------|:----:|:-----|
| 书名+作者 | 是 | 《冰河末世》记忆的海 |
| 字数+状态 | 是 | 2876章连载中 / 102万字完结 |
| 核心标签 | 是 | 末世+重生+空间+囤货 |
| 简介 | 是 | 作品页100字以上简介 |
| 在读/热度数据 | 尽量 | 50万+在读 |
| 前3章内容/片段 | 尽量 | 搜索试读或读者引用的原文 |
| 读者评论（5条+） | 尽量 | 番茄书评区 |
| 结构/节奏相关信息 | 尽量 | 有人说"前期快后期慢"等 |

**达标判定**：必需项100% + 尽量项≥50% = 作品信息达标

## 4. 搜索失败/信息不足降级策略

| 情况 | 降级方案 |
|:-----|:---------|
| 平台封锁正文 | 使用 passage_extractor.py --mode infer 处理搜索片段 |
| 找不到热门作品 | 使用 SKILL 内置类型知识库 + genre-taxonomy.md 兜底 |
| 对标作品信息不足 | 降低该作品的分析深度，标注素材完整度 |
| 读者评论不足 | 标注推断置信度，避免伪装为实证结论 |
| 无前3章内容 | 跳过范文精析环节，AI 推断结构特征并标注 |

## 5. 素材完整度评估公式

```
素材完整度 = (已获取的信息类别数 / 总信息类别数) × 100%

总信息类别 = 8（上表全部类别）
5级表示：●=100% / ○=0%，每级20%

●●●●● = 80-100%（6-8类已获取）
●●●●○ = 60-79%（5类）
●●●○○ = 40-59%（3-4类）
●●○○○ = 20-39%（2类）
●○○○○ = 0-19%（0-1类）
```

## 6. 范文精析降级说明

当无法获取原文时，范文精析流程改为：

```
标准流程：获取原文 → passage_extractor.py --mode opening/climax → 精确技法标注
降级流程：收集搜索片段/简介/书评 → passage_extractor.py --mode infer → 推断技法标注 + 置信度
```

降级流程下的技法分析**必须**在报告中附加推断标注：
```
⚠️推断·置信度：高/中/低
⚠️推断·原因：基于搜索片段/仅获取简介/仅获取书评
```
