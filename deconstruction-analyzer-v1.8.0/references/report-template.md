# 拆书报告模板

> 本文件约束拆书报告的输出格式，确保与 multi-role-novel-iteration 兼容

---

## 报告结构

拆书报告由以下固定部分组成：

### 1. YAML 元信息头

```yaml
分析时间: "YYYY-MM-DD HH:MM"
分析对象: "类型分析 - 大类 > 小类 > 子类"
目标类型: "大类名"
子类型: "小类 / 子类"
字数区间: "15000-25000字"
适用写作模式: "web"  # 由 TYPE_TO_WRITING_MODE 按大类推导（玄幻→fantasy, 悬疑/推理→mystery, 言情→romance, 科幻→scifi 等）
类型匹配: "匹配描述"
用户意图: "全面分析"
交叉类型: ""  # 由 type_router 自动检测，触发时格式为"大类A + 大类B"（如"末世/废土 + 都市"）；关键词推断时secondary来源为推断
匹配状态: "✅ 已匹配 / ⚠️ 未匹配"
素材完整度: "●●●●●"  # 5级，计算公式：(已获取信息类别数/8)×100%，详见搜索策略SOP第5节；●=100%/○=0%，每级20%
字数档位节奏公式:  # 由 type_router formula_by_wordcount 生成，供 multi-role 按目标字数自适应；无数据时留空 {}
  "2万字":
    开篇: "xxx"
    爽点节奏: "xxx"
    字数分配: "xxx"
    备注: "xxx"
  "5万字":
    开篇: "xxx"
    爽点节奏: "xxx"
    字数分配: "xxx"
    备注: "xxx"
  "20万字+":
    开篇: "xxx"
    爽点节奏: "xxx"
    字数分配: "xxx"
    备注: "xxx"
协议版本: "1.5.0"
```

### 2. 分析维度权重

> 分析维度权重： 结构 ⭐⭐⭐⭐ / 节奏 ⭐⭐⭐⭐⭐ / ...

> 💡 额外分析维度：xxx、xxx

### 3. 九章节正文

| 序号 | 章节标题 | 标记 | 最少子项数 | 必含字段 |
|:----:|:---------|:----:|:----------:|:---------|
| 一 | 类型总览 | 📋 | 4 | 市场热力值、商业化建议、核心类型特征、对标作品 |
| 二 | 结构模板 | 🏗️ | 3 | 黄金三章、全篇结构、字数与节奏 |
| 三 | 爽点节奏 | 📈 | 4 | 爽点类型分布、节奏对标、情绪曲线、钩子设计 |
| 四 | 人物原型 | 👥 | 3 | 主角原型、配角模板、人设模板 |
| 五 | 开篇钩子 | 🎣 | 2 | 开篇公式、自检清单 |
| 六 | 金手指 | 💎 | 3 | 金手指分布、创新机会、设计原则 |
| 七 | 常见雷区 | ⚠️ | 3 | 雷区清单、负面反馈、创新机会 |
| 八 | 多角色速查表 | 📊 | — | 大纲师 |
| 九 | 文风DNA与量化指标 | 🧬 | 2 | 文风DNA标签、量化指标卡 |

> **必含字段精确名称**：上述"必含字段"列中的字段名**必须作为子标题或正文中出现的精确字符串**，校验器会按此名称检测。例如第九章必须出现"量化指标卡"（而非"量化指标汇总"或"量化指标"），第五章必须出现"开篇公式"和"自检清单"。

> 第一章可附「### 平台适配建议」子节（由 AI 基于搜索数据填充分析，`platform_matcher.py` 已迁移至 multi-role-novel-iteration）

### 4. 量化指标卡（每章节末尾附）

每个分析章节末尾必须附「量化指标卡」，格式如下：

```
📊 **量化指标卡**
| 指标 | 数值 | 健康阈值 | 状态 |
|------|------|----------|:----:|
| xxx | xxx | xxx | ✅/⚠️/❌ |
```

各章节对应指标：

| 章节 | 必含指标 | 健康阈值 |
|------|---------|---------|
| 二·结构模板 | 过渡型章节占比 | ≤15% |
| 三·爽点节奏 | 爽点密度 | 前20章每3章≥1次、全文每5章≥1次 |
| 四·人物原型 | 配角戏份比 / 对话辨识度 | ≤主角30% / ≥60%（去人名测试） |
| 六·金手指 | 设定说明密度 | ≤500字/次 |
| 五·开篇钩子 | 黄金三章信息密度 | ≥5条新信息/章 |
| 七·常见雷区 | 反转铺垫比 / 伏笔回收率 | 铺垫≥3处 / 核心≥90% |

### 5. 可视化图谱（Mermaid格式）

报告必须包含 **3类一级图谱** 和推荐包含 **2类二级图谱**：

**一级图谱（必含）**：

| 图谱 | 对应章节 | Mermaid图类型 |
|------|---------|--------------|
| 情绪节奏曲线图 | 三·爽点节奏 | xychart-beta |
| 人物关系拓扑图 | 四·人物原型 | graph |
| 故事结构框架图 | 二·结构模板 | flowchart |

**二级图谱（推荐）**：

| 图谱 | 对应章节 | Mermaid图类型 |
|------|---------|--------------|
| 爽点密度热力图 | 三·爽点节奏 | timeline |
| 冲突升级阶梯图 | 七·常见雷区 | flowchart |

图谱数据必须来自实际拆解结果，不可编造。标注规范：🔥高潮=红色、⚡爽点=橙色、🪝钩子=蓝色、🔮伏笔=紫色。

### 6. 推断标注规则

基于 web 搜索片段而非完整正文的分析，必须标注推断透明度：

```
⚠️推断·置信度：高/中/低
⚠️推断·原因：基于web搜索片段/仅获取书籍信息/仅获取书评
```

推断性结论不得伪装为实证分析。素材完整度≤60%时，宏观结论必须附加推断标注。

### 7. 可选附录：深度切片

当用户需要更深入的单本分析时，可在报告正文之后附加「附录·深度切片」：

- **黄金三章逐段分析**：逐段标注功能（引入/展示/冲突/信息/钩子）
- **高潮场景逐句拆解**：标注"铺垫→爆发→余韵"技法
- **对话去名辨识度测试**：5段关键对话的去名测试详细结果

此附录为可选章节，不影响主报告9章结构。

### 8. 引用标记格式

```
[拆书参考] 根据报告...
[拆书参考-七2] 避雷：...
[拆书参考-九1] 文风DNA参考：...
```

### 9. 输出路径

```
{项目目录}/deconstruction_report/report.md
```
